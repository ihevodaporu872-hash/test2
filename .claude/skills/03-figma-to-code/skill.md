# Skill: Figma → Code via MCP

## Цель
Максимально точно переносить макеты из Figma в Next.js + Tailwind + shadcn, сохраняя дизайн-токены и адаптивность.

## Когда использовать MCP Figma
- Есть ссылка на Frame/Section/Component.
- Нужно восстановить spacing/typography/colors/radii.
- Нужно разобрать компонентную структуру.

## Процесс (строго)
1) Через Figma MCP получить:
   - размеры/constraints,
   - тексты и стили,
   - компоненты и варианты,
   - токены (цвета, шрифты, размеры).
2) Сопоставить с design system:
   - цвета → tailwind tokens,
   - размеры → spacing scale,
   - радиусы → `--radius`/rounded tokens.
3) Собрать layout:
   - grid/flex, контейнеры, responsive breakpoints.
4) Перевести в переиспользуемые компоненты:
   - секции в `components/sections`
   - карточки/повторяющиеся блоки в `components/shared`
5) Проверить:
   - mobile/tablet/desktop,
   - a11y,
   - отсутствие “пиксельного рабства”, если оно ломает адаптив.

## Анти-паттерны
- Копировать абсолютные px-величины без нужды.
- Делать “компонент на один раз”, если он повторяется.
- Игнорировать состояния компонентов.

## DoD
- Визуально близко к макету.
- Код не хрупкий и не переусложнён.
- Токены вынесены, а не размазаны по классам.
